# Orchestration CLAUDE.md

## 1. 목적
사용자 요청을 해석하고 실행 흐름을 관리한다.

## 2. 핵심 역할
- 의도 분석
- tool 선택
- 실행 순서 제어
- 결과 요약

## 3. 설계 원칙
- Tool 기반 실행
- 정책 중심 제어
- 확장 가능한 구조

## 4. 주요 기능
- 테스트 스크립트 생성
- 테스트 실행 요청
- 결과 분석
- 실패 원인 설명

## 5. Tool 사용 규칙
- 모든 실행은 tool 통해 수행
- 직접 외부 호출 금지

## 6. 정책
- 승인 필요 작업 정의
- 위험 작업 차단

## 7. 금지사항
- 직접 Jenkins 호출 금지
- 직접 DB 접근 금지
- 임의 코드 실행 금지

## 8. 실행 흐름
1. 사용자 입력 수신
2. 의도 분석
3. tool 선택
4. 실행
5. 결과 반환

## 9. 에러 처리
- tool 실패 시 재시도
- fallback 전략 적용

## 10. 상태 관리
- 세션 기반 상태 유지
- 작업 상태 추적

## 11. 로그
- tool 호출 기록
- 실행 결과 저장

## 12. 보안
- 권한 기반 tool 실행
- 사용자 격리

## 13. 확장성
- tool 추가 용이
- 정책 변경 용이

## 14. 테스트
- 시나리오 기반 테스트
- 실패 케이스 검증

## 15. 유지보수
- tool 인터페이스 안정성 유지
- 정책 업데이트 관리
