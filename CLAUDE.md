# Windows Terminal Client CLAUDE.md

## 1. 목적
이 클라이언트는 사용자 인터페이스(UI) 역할만 수행한다.
모든 실제 실행 로직은 서버에서 처리된다.

## 2. 핵심 역할
- 사용자 입력 수집
- 서버 세션 연결 유지
- 응답 스트리밍 출력
- 승인/확인 입력 처리

## 3. 설계 원칙
- Thin Client 유지
- 상태는 서버 기준
- 로컬 로직 최소화

## 4. 기능 범위
- 로그인 / 인증
- 세션 시작 / 종료
- 명령 입력
- 결과 표시

## 5. 금지사항
- Jenkins 직접 호출 금지
- Git 직접 접근 금지
- DB 접근 금지
- 테스트 실행 로직 포함 금지
- 로컬 파일 시스템 변경 금지

## 6. 보안 원칙
- 토큰은 메모리에서만 유지
- 로그에 민감정보 저장 금지
- 세션 타임아웃 필수

## 7. UI 규칙
- 결과는 스트리밍으로 표시
- 긴 작업은 진행 상태 표시
- 오류는 사용자 친화적으로 출력

## 8. 통신 방식
- HTTPS 기반 API 호출
- WebSocket 또는 SSE 사용 가능

## 9. 상태 관리
- 서버에서 내려주는 상태만 사용
- 로컬 캐시 최소화

## 10. 에러 처리
- 재시도 로직 포함
- 네트워크 오류 대응
- 사용자에게 명확한 메시지 제공

## 11. 로그 정책
- 디버그 로그 제한적 사용
- 사용자 행동 로그 최소화

## 12. 확장성
- 기능 추가는 서버 중심
- 클라이언트 변경 최소화

## 13. 배포
- PyInstaller로 exe 패키징
- 자동 업데이트 최소화

## 14. 테스트
- API mock 기반 테스트
- UI 단위 테스트

## 15. 유지보수
- 버전 호환성 유지
- API 스펙 기반 개발
